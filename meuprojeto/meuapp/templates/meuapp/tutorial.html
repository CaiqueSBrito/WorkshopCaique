O que é uma API REST?

Entendendo o conceito antes de codar

API significa Application Programming Interface. Pense como um garçom num restaurante: você (o cliente / frontend) faz um pedido → o garçom (a API) leva para a cozinha (o backend / banco de dados) → e traz o resultado de volta.

Uma REST API (Representational State Transfer) é um estilo de arquitetura onde o servidor expõe recursos (dados) via URLs, e o cliente manipula esses recursos usando os verbos HTTP.

🌐
Recurso (Resource)
Qualquer dado: um produto, usuário, artigo. Cada recurso tem uma URL única. Ex: /api/produtos/1/
📨
Request / Response
O cliente envia uma requisição (request) e recebe uma resposta (response) em formato JSON.
🔑
Stateless
Cada requisição é independente. O servidor não lembra de requisições anteriores — você precisa enviar tudo necessário em cada chamada.
📦
JSON
O formato padrão para troca de dados. Leve, legível por humanos e perfeito para JavaScript.
💡
Exemplo do dia a dia: Quando você abre o Instagram no celular, o app faz chamadas para a API do Instagram pedindo as fotos, curtidas e comentários. O Django REST Framework seria o "servidor por trás" que responde a essas chamadas.

Verbos HTTP & Status Codes

A linguagem de comunicação da web

Verbos (Métodos HTTP)

Cada ação que você faz com dados usa um verbo diferente:

Método Ação Exemplo Analogia
GET Ler / buscar dados GET /api/produtos/ Olhar o cardápio
POST Criar novo dado POST /api/produtos/ Fazer um pedido
PUT Atualizar tudo PUT /api/produtos/1/ Trocar o prato inteiro
PATCH Atualizar parcialmente PATCH /api/produtos/1/ Tirar só a cebola
DELETE Deletar dado DELETE /api/produtos/1/ Cancelar o pedido

Códigos de Status

✅
2xx — Sucesso
200 OK · 201 Criado · 204 Sem conteúdo (delete)
↩️
3xx — Redirecionamento
301 Movido permanentemente · 304 Não modificado
❌
4xx — Erro do cliente
400 Dado inválido · 401 Não autenticado · 403 Sem permissão · 404 Não encontrado
💥
5xx — Erro do servidor
500 Erro interno · 503 Serviço indisponível

Models — Os Dados

Definindo as tabelas do banco de dados

O Model define a estrutura do seu dado. Cada classe Python vira uma tabela no banco de dados. Cada atributo vira uma coluna.

python — produtos/models.py 
from django.db import models
from django.contrib.auth.models import User

class Categoria(models.Model):
    nome = models.CharField(max_length=100)
    descricao = models.TextField(blank=True)

    def __str__(self):
        return self.nome

    class Meta:
        verbose_name_plural = 'Categorias'


class Produto(models.Model):
    # Choices para o campo status
    STATUS_CHOICES = [
        ('ativo', 'Ativo'),
        ('inativo', 'Inativo'),
        ('esgotado', 'Esgotado'),
    ]

    nome = models.CharField(max_length=200)
    descricao = models.TextField(blank=True)
    preco = models.DecimalField(max_digits=10, decimal_places=2)
    estoque = models.IntegerField(default=0)
    status = models.CharField(max_length=10, choices=STATUS_CHOICES, default='ativo')

    # ForeignKey = relação de 1 categoria → muitos produtos
    categoria = models.ForeignKey(
        Categoria,
        on_delete=models.CASCADE,   # se deletar categoria, deleta produtos
        related_name='produtos'
    )

    # O criador do produto (usuário logado)
    criado_por = models.ForeignKey(
        User,
        on_delete=models.SET_NULL,
        null=True, blank=True
    )

    criado_em = models.DateTimeField(auto_now_add=True)  # preenche automaticamente
    atualizado_em = models.DateTimeField(auto_now=True)   # atualiza automaticamente

    def __str__(self):
        return self.nome

    class Meta:
        ordering = ['-criado_em']  # Mais recentes primeiro

Serializers — A Tradução

Convertendo Python ↔ JSON

O Serializer é como um "tradutor". Ele converte objetos Python (do banco de dados) em JSON para enviar ao cliente — e também valida e converte JSON de volta em objetos Python quando recebe dados.

💡
Imagine que você tem um objeto Produto Python. O serializer "serializa" ele para {"nome": "Camiseta", "preco": "49.90", ...}. E quando o JS envia um JSON, o serializer valida se está tudo certo antes de salvar.
python — Serializer manual 
from rest_framework import serializers

# Serializer feito "na mão" — raramente usado
class ProdutoSerializer(serializers.Serializer):
    id = serializers.IntegerField(read_only=True)
    nome = serializers.CharField(max_length=200)
    preco = serializers.DecimalField(max_digits=10, decimal_places=2)

    def create(self, validated_data):
        return Produto.objects.create(**validated_data)

    def update(self, instance, validated_data):
        instance.nome = validated_data.get('nome', instance.nome)
        instance.preco = validated_data.get('preco', instance.preco)
        instance.save()
        return instance
python — ModelSerializer (recomendado) 
from rest_framework import serializers
from .models import Produto, Categoria

class ProdutoSerializer(serializers.ModelSerializer):

    # Validação de campo único: validate_
    def validate_preco(self, value):
        if value <= 0:
            raise serializers.ValidationError('Preço deve ser maior que zero')
        return value

    # Validação entre múltiplos campos
    def validate(self, data):
        if data['estoque'] > 0 and data['status'] == 'esgotado':
            raise serializers.ValidationError(
                'Produto com estoque não pode ser esgotado'
            )
        return data

    class Meta:
        model = Produto
        fields = '__all__'

Views & ViewSets

A lógica que responde às requisições

As Views são funções ou classes que recebem uma requisição HTTP e retornam uma resposta. O DRF oferece várias formas de criar views, da mais manual à mais automática.

Mais verboso, mas dá total controle. Ideal para lógica complexa.

python — produtos/views.py 
from rest_framework.views import APIView
from rest_framework.response import Response
from rest_framework import status
from .models import Produto
from .serializers import ProdutoSerializer

class ProdutoListView(APIView):
    """GET /api/produtos/  e  POST /api/produtos/"""

    def get(self, request):
        produtos = Produto.objects.all()
        serializer = ProdutoSerializer(produtos, many=True)
        return Response(serializer.data)

    def post(self, request):
        serializer = ProdutoSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save(criado_por=request.user)
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)


class ProdutoDetailView(APIView):
    """GET, PUT, DELETE /api/produtos/{id}/"""

    def get_object(self, pk):
        try:
            return Produto.objects.get(pk=pk)
        except Produto.DoesNotExist:
            return None

    def get(self, request, pk):
        produto = self.get_object(pk)
        if not produto:
            return Response(status=status.HTTP_404_NOT_FOUND)
        serializer = ProdutoSerializer(produto)
        return Response(serializer.data)

    def put(self, request, pk):
        produto = self.get_object(pk)
        if not produto:
            return Response(status=status.HTTP_404_NOT_FOUND)
        serializer = ProdutoSerializer(produto, data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

    def delete(self, request, pk):
        produto = self.get_object(pk)
        if not produto:
            return Response(status=status.HTTP_404_NOT_FOUND)
        produto.delete()
        return Response(status=status.HTTP_204_NO_CONTENT)

Mixins prontos que eliminam código repetitivo. Usa herança para combinar funcionalidades.

python — Generic Views 
from rest_framework import generics
from rest_framework.permissions import IsAuthenticatedOrReadOnly
from .models import Produto
from .serializers import ProdutoSerializer

# Lista + Criação
class ProdutoListCreateView(generics.ListCreateAPIView):
    queryset = Produto.objects.all()
    serializer_class = ProdutoSerializer
    permission_classes = [IsAuthenticatedOrReadOnly]

    # Customizar o create para salvar o usuário
    def perform_create(self, serializer):
        serializer.save(criado_por=self.request.user)


# Detalhe + Atualização + Deleção
class ProdutoDetailView(generics.RetrieveUpdateDestroyAPIView):
    queryset = Produto.objects.all()
    serializer_class = ProdutoSerializer

# Classes disponíveis:
# ListAPIView          → só GET lista
# CreateAPIView        → só POST
# RetrieveAPIView      → só GET detalhe
# UpdateAPIView        → só PUT/PATCH
# DestroyAPIView       → só DELETE
# ListCreateAPIView    → GET lista + POST
# RetrieveUpdateDestroyAPIView → GET detalhe + PUT/PATCH + DELETE

⚡ A forma mais poderosa e recomendada. Um ViewSet reune todas as operações numa única classe, e o Router cria as URLs automaticamente.

python — ViewSet (recomendado) 
from rest_framework import viewsets, filters
from rest_framework.decorators import action
from rest_framework.response import Response
from django_filters.rest_framework import DjangoFilterBackend
from .models import Produto
from .serializers import ProdutoSerializer, ProdutoDetalheSerializer

class ProdutoViewSet(viewsets.ModelViewSet):
    """
    ViewSet completo — gera automaticamente:
    GET    /api/produtos/          → list()
    POST   /api/produtos/          → create()
    GET    /api/produtos/{id}/     → retrieve()
    PUT    /api/produtos/{id}/     → update()
    PATCH  /api/produtos/{id}/     → partial_update()
    DELETE /api/produtos/{id}/     → destroy()
    """
    queryset = Produto.objects.all()
    serializer_class = ProdutoSerializer

    # Filtros, busca e ordenação
    filter_backends = [DjangoFilterBackend, filters.SearchFilter, filters.OrderingFilter]
    filterset_fields = ['status', 'categoria']
    search_fields = ['nome', 'descricao']
    ordering_fields = ['preco', 'criado_em']

    # Usar serializer diferente para detalhe vs lista
    def get_serializer_class(self):
        if self.action in ['retrieve']:
            return ProdutoDetalheSerializer
        return ProdutoSerializer

    # Action customizada: GET /api/produtos/em-destaque/
    @action(detail=False, methods=['get'], url_path='em-destaque')
    def em_destaque(self, request):
        destaque = self.queryset.filter(status='ativo')[:5]
        serializer = self.get_serializer(destaque, many=True)
        return Response(serializer.data)

Autenticação

JWT Token — o padrão moderno

Autenticação é o processo de verificar quem está fazendo a requisição. O método mais usado em APIs modernas é o JWT (JSON Web Token).

🔑
Como funciona o JWT: O usuário faz login e recebe 2 tokens: access (expira em 5 min) e refresh (expira em dias). O access é enviado em toda requisição protegida. Quando expira, usa o refresh para pegar um novo access.
bash 
pip install djangorestframework-simplejwt
python — settings.py

Usando o Token

javascript — Autenticação JWT 
// 1. Fazer login
const login = async (username, password) => {
  const response = await fetch('/api/auth/login/', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password })
  });

  const data = await response.json();
  // data = { access: "eyJ...", refresh: "eyJ..." }

  // Salvar no localStorage
  localStorage.setItem('access_token', data.access);
  localStorage.setItem('refresh_token', data.refresh);
};

// 2. Fazer requisição autenticada
const getToken = () => localStorage.getItem('access_token');

const authFetch = (url, options = {}) => {
  return fetch(url, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${getToken()}`,  // Token no header!
      ...options.headers,
    }
  });
};

from rest_framework.permissions import BasePermission

class IsDonoDoProduto(BasePermission):
    """Só o criador pode editar/deletar"""

    def has_object_permission(self, request, view, obj):
        # Leitura sempre permitida
        if request.method in ['GET', 'HEAD', 'OPTIONS']:
            return True

        # Escrita só para o criador
        return obj.criado_por == request.user


# Usar na view:

JavaScript — Fetch API

Consumindo a API no frontend sem bibliotecas

A Fetch API é nativa do JavaScript moderno. Não precisa instalar nada — funciona direto no navegador.

javascript — Configuração base 
// Configuração base da API
const API_BASE = 'http://localhost:8000/api';
const TOKEN = () => localStorage.getItem('access_token');

// Headers padrão com autenticação
const headers = (withAuth = true) => ({
  'Content-Type': 'application/json',
  ...(withAuth && TOKEN() ? { 'Authorization': `Bearer ${TOKEN()}` } : {})
});

// Handler de erros reutilizável
const handleResponse = async (response) => {
  if (!response.ok) {
    const error = await response.json().catch(() => ({}));
    throw new Error(JSON.stringify(error) || `HTTP ${response.status}`);
  }
  // 204 No Content não tem body
  if (response.status === 204) return null;
  return response.json();
};
javascript — GET 
// Listar todos os produtos
const listarProdutos = async () => {
  try {
    const response = await fetch(`${API_BASE}/produtos/`, {
      method: 'GET',
      headers: headers()
    });
    const data = await handleResponse(response);
    console.log(data);
    // data.results → array de produtos (com paginação)
    // data.count → total de registros
    // data.next → URL da próxima página
    return data;
  } catch (err) {
    console.error('Erro ao listar:', err.message);
  }
};

// Buscar produto específico
const getProduto = async (id) => {
  const response = await fetch(`${API_BASE}/produtos/${id}/`, {
    headers: headers()
  });
  return handleResponse(response);
};

// Com filtros e busca
const buscarProdutos = async (busca, status) => {
  const params = new URLSearchParams({
    ...(busca && { search: busca }),
    ...(status && { status }),
    ordering: 'preco'
  });
  const response = await fetch(`${API_BASE}/produtos/?${params}`, {
    headers: headers()
  });
  return handleResponse(response);
};
javascript — POST 
// Criar novo produto
const criarProduto = async (dadosProduto) => {
  try {
    const response = await fetch(`${API_BASE}/produtos/`, {
      method: 'POST',
      headers: headers(),
      body: JSON.stringify(dadosProduto)
    });

    console.log('Criado!', novoProduto);
    return novoProduto;

    console.error('Erro ao criar:', err.message);
    // err.message pode conter os erros de validação do DRF
    // Ex: '{"nome":["Este campo é obrigatório."]}'
  }
};

// Exemplo de uso
criarProduto({
  nome: 'Camiseta Básica',

  preco: '49.90',
  estoque: 100,

javascript — PUT & PATCH 
// PUT = atualiza tudo (envia todos os campos)
const atualizarProduto = async (id, dados) => {
  const response = await fetch(`${API_BASE}/produtos/${id}/`, {
    method: 'PUT',
    headers: headers(),
    body: JSON.stringify(dados)
  });
  return handleResponse(response);
};

// PATCH = atualiza só os campos enviados
const atualizarParcial = async (id, camposParaAtualizar) => {
  const response = await fetch(`${API_BASE}/produtos/${id}/`, {
    method: 'PATCH',
    headers: headers(),
    body: JSON.stringify(camposParaAtualizar)

javascript — Configuração Axios 
// Cria instância configurada
const api = axios.create({
  baseURL: 'http://localhost:8000/api',
  headers: { 'Content-Type': 'application/json' }
});

// Interceptor: adiciona o token em TODA requisição automaticamente
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Interceptor: trata erros globalmente
api.interceptors.response.use(
  (response) => response.data,   // Retorna só os dados
  async (error) => {
    if (error.response?.status === 401) {
      // Token expirado → tenta renovar
      const refresh = localStorage.getItem('refresh_token');
      try {
        const { data } = await axios.post('/api/auth/refresh/', { refresh });
        localStorage.setItem('access_token', data.access);
        // Reenviar request original
        return api(error.config);
      } catch {
        // Refresh também expirou → redireciona para login
        window.location.href = '/login';
      }
    }
    return Promise.reject(error);
  }
);

// Uso simplificado
const produtos = await api.get('/produtos/');

CRUD Completo em JS

Exemplo real de um módulo completo

javascript — produtos.js (módulo completo) 
// === MÓDULO DE PRODUTOS ===
class ProdutosAPI {
  constructor(baseURL) {
    this.url = `${baseURL}/produtos`;
  }

  async listar({ pagina = 1, busca = '', status = '' } = {}) {
    const params = new URLSearchParams({ page: pagina, search: busca });
    if (status) params.set('status', status);

    const res = await fetch(`${this.url}/?${params}`, {
      headers: { 'Authorization': `Bearer ${getToken()}` }
    });
    return res.json();
  }

  async buscar(id) {
    const res = await fetch(`${this.url}/${id}/`, {
      headers: { 'Authorization': `Bearer ${getToken()}` }
    });
    if (!res.ok) throw new Error('Produto não encontrado');
    return res.json();
  }

  async criar(dados) {
    const res = await fetch(`${this.url}/`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${getToken()}`
      },
      body: JSON.stringify(dados)
    });

    if (res.status === 400) {
      const erros = await res.json();
      throw new Error(JSON.stringify(erros));  // Erros de validação
    }

    return res.json();
  }

  async atualizar(id, dados, parcial = true) {
    const res = await fetch(`${this.url}/${id}/`, {
      method: parcial ? 'PATCH' : 'PUT',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${getToken()}`
      },
      body: JSON.stringify(dados)
    });
    return res.json();
  }

  async deletar(id) {
    const res = await fetch(`${this.url}/${id}/`, {
      method: 'DELETE',
      headers: { 'Authorization': `Bearer ${getToken()}` }
    });
    return res.status === 204;
  }
}

// === USO NO FRONTEND ===
const produtosAPI = new ProdutosAPI('http://localhost:8000/api');

// Renderizar lista de produtos
async function renderizarProdutos() {
  const lista = document.getElementById('lista-produtos');
  lista.innerHTML = '<p>Carregando...</p>';

  try {
    const data = await produtosAPI.listar();

    lista.innerHTML = data.results.map(p => `
      <div class="produto-card" data-id="${p.id}">
        <h3>${p.nome}</h3>
        <p>R$ ${parseFloat(p.preco).toFixed(2)}</p>
        <span class="status ${p.status}">${p.status}</span>
        <button onclick="editarProduto(${p.id})">Editar</button>
        <button onclick="confirmarDelete(${p.id}, '${p.nome}')">Deletar</button>
      </div>
    `).join('');
  } catch (err) {
    lista.innerHTML = `<p class="erro">Erro: ${err.message}</p>`;
  }
}

import django_filters
from .models import Produto

class ProdutoFilter(django_filters.FilterSet):
    # Filtros de intervalo de preço
    preco_min = django_filters.NumberFilter(field_name='preco', lookup_expr='gte')
    preco_max = django_filters.NumberFilter(field_name='preco', lookup_expr='lte')

    # Filtro por nome (contém, case-insensitive)
    nome = django_filters.CharFilter(lookup_expr='icontains')

    class Meta:
        model = Produto
        fields = ['status', 'categoria', 'nome']

async function carregarPagina(pagina = 1) {
  const res = await fetch(`/api/produtos/?page=${pagina}`);
  const { count, next, previous, results } = await res.json();

  renderizarProdutos(results);

function renderizarPaginacao(atual, total, next, previous) {
  const nav = document.getElementById('paginacao');
  nav.innerHTML = `
    <button ${!previous ? 'disabled' : ''} onclick="carregarPagina(${atual-1})">
      ← Anterior
    </button>
    <span>Página ${atual} de ${total}</span>
    <button ${!next ? 'disabled' : ''} onclick="carregarPagina(${atual+1})">
      Próxima →
    </button>
  `;
}
